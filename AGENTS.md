# AGENTS

## Обязательный процесс поставки (Spec-Driven Development)

Проект обязан следовать Spec-Driven Development для всех фич и изменений.

1. Сначала спецификация
2. Потом план
3. Потом реализация
4. Потом проверка

Любая реализация запрещена, пока спецификация и план не согласованы.

## Обязательные рабочие правила

- Всегда начинать со спецификации, затем делать план, затем реализацию, затем проверку.
- Использовать русский язык для всей проектной коммуникации и документации по умолчанию.
- Сохранять все промпты и результаты в REPORT.md по ходу работы, не откладывая на конец.
- Не пропускать ни одного требования из REQUIREMENTS.md.
- Если требования конфликтуют, сначала явно обновить спецификацию, потом менять реализацию.

## Минимальный контрольный чеклист по каждой задаче

- Спецификация существует и актуальна.
- План соответствует текущей спецификации.
- Изменения в коде сопоставлены с пунктами спецификации.
- Тесты и проверки подтверждают критерии приемки.
- Лог промптов обновлен в REPORT.md.
